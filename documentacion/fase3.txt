EXPLICACIÓN PROFESIONAL

ini_set() y error_reporting()

ini_set('display_errors', 1);
error_reporting(E_ALL);

¿Qué hacen?

Muestran errores durante desarrollo.

¿Por qué es importante?

Cuando programamos necesitamos:

detectar errores,
depurar,
entender fallos.

⚠️ En producción esto se desactiva por seguridad.

session_start()

session_start();

¿Qué hace?

Inicia el sistema de sesiones PHP.

Gracias a esto podremos:

mantener usuarios logueados,
guardar datos temporales,
manejar autenticación.

BASE_PATH

define('BASE_PATH', dirname(__DIR__));

¿Qué hace?

Define la raíz del proyecto.

Ejemplo:

C:/xampp/htdocs/taskflow

¿Por qué se usa?

Evita escribir rutas manuales enormes.

Autoload

require_once BASE_PATH . '/app/core/Autoload.php';

¿Qué es?

Sistema de carga automática de clases.

¿Qué problema resuelve?

Sin autoload tendrías que hacer:

require_once 'controllers/HomeController.php';
require_once 'models/User.php';
require_once 'core/Router.php';

TODO el tiempo.

Eso destruye mantenimiento.

2. AUTOLOAD

Archivo:
/app/core/Autoload.php

Código

<?php

class Autoload
{
    public function __construct()
    {
        spl_autoload_register([$this, 'loadClass']);
    }

    private function loadClass($class)
    {
        $paths = [
            BASE_PATH . '/app/controllers/',
            BASE_PATH . '/app/models/',
            BASE_PATH . '/app/core/',
        ];

        foreach ($paths as $path) {

            $file = $path . $class . '.php';

            if (file_exists($file)) {
                require_once $file;
                return;
            }
        }
    }
}
EXPLICACIÓN PROFESIONAL

spl_autoload_register()

¿Qué hace?

Le dice a PHP:

"Cuando una clase sea utilizada,

búscala automáticamente."

Ejemplo

Cuando escribamos:

new Router();

PHP automáticamente buscará:

/app/core/Router.php

¿Qué ventaja tiene?

✅ Organización
✅ Escalabilidad
✅ Menos require_once
✅ Código limpio
✅ Arquitectura profesional
